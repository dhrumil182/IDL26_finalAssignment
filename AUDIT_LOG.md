# Incident Audit Log — Operation Cyber-Histology

This log lists every bug we found in the recovered codebase, what caused it, how we fixed it, and the commit that has the fix.

## Section 1 — Pipeline Reconstruction

| # | File | What Went Wrong | Why It Happened | How We Fixed It | Git Commit Hash |
|---|------|--------------------------|------------|--------------------------|------------------|
| 1 | (repo root) | Couldn't reinstall the right packages | The environment wipe deleted the original requirements file, and nothing was left in the drive cache | Wrote a new `requirements.txt` listing exactly what the code imports | `d4ab76d32822534d64b1ebc47ffe96994f324302` |
| 2 | fit.py | Training loss never went down; the model just didn't learn | `train_one_epoch` never called `optimizer.zero_grad()`, so gradients kept piling up batch after batch instead of resetting, which messed up every update | Added `self.optimizer.zero_grad()` right before `loss.backward()` | `f5cb625537ce9b086c509d1d9c0135ff7485e7f8` |
| 3 | fit.py | A variable name was silently breaking Python's built-in `sum()` | The batch counter was named `sum`, which shadowed the built-in `sum()` function for the rest of that function | Renamed it to `total` | `712f3bd0ff748a58b51dd43a1d1303798e44ac49` |
| 4 | data.py | Every dataset threw a `FileNotFoundError` on startup | The loader was looking for `{data}_data.pt`, but the recovered files are actually named `{data}.pt` | Fixed the filename pattern | `017c13c26a6ed5a48eac0f04821c142bb73b591b` |
| 5 | data.py | Validation accuracy looked suspiciously close to training accuracy, inflating the benchmark numbers | The validation set was just a slice of the same tensor that was also handed to the training loader in full, so the model was training on its own validation data | Swapped the slice for `torch.utils.data.random_split`, so train and validation sets no longer overlap, with a fixed seed for reproducibility | `d8f49d1e0645711c8bca6bd9420b213a3a136997` |
| 6 | data.py | Label tensors had different shapes depending on the dataset, which broke `CrossEntropyLoss` | Some datasets store labels as `(N, 1)` instead of `(N,)`, and nothing normalized that before use | Added `.squeeze(1)` to the label tensors before building the dataset | `170c56ab7136310ce9553abee8fcce7c8dcf395e` |
| 7 | data.py | The pipeline only worked for one dataset because `CHANNELS`/`NUM_CLASSES` had to be typed into the config by hand | Instead of reading the data's actual shape, the code expected someone to manually keep the config in sync with each dataset | `get_loaders` now figures out `num_channels`, `num_classes`, and `input_size` directly from the data | `d447a457ed3e15e4fbdb8736a612bd99310524b2` |
| 8 | models.py | Bigger `VGGBlock` stages (with 3 conv layers) crashed with channel-mismatch errors on the 2nd or 3rd layer | The code never updated `current_in_channels` after the first conv layer, so later layers kept using the block's original input size instead of the previous layer's output size | Now tracks and updates `current_in_channels` on every loop iteration | `95f2e617a87b813766d74c45836ce55e8f38d2f2` |
| 9 | models.py | VGG's 1x1 convolutions at the end of a block were changing the image size instead of leaving it alone, which broke the fixed-size flatten step later on | Padding was hardcoded to `1` for every kernel size, even the 1x1 ones, which actually need `0` padding | Padding is now picked based on kernel size (`0` for 1x1, `1` otherwise) | `95f2e617a87b813766d74c45836ce55e8f38d2f2` |
| 10 | models.py | Every model trained as if it had no activation function at all — loss flattened out immediately and accuracy stayed near random guessing | A leftover `activation_str = "Identity"` placeholder was hardcoded at the module level and never replaced with `config["ACTIVATION"]`; `ResNet18` used this placeholder directly, and `VGGBlock`/`AlexNet`/`VGG16` had `nn.ReLU` hardcoded too, so the config setting was ignored either way | Removed the leftover placeholder — every model now reads `activation_str` from its arguments and builds the actual activation the config asked for | `d137e8f4d6be48b0c59838b4e25b6fd07372a3b6` |
| 11 | models.py | `AlexNet` only worked with exactly 3 input channels and 11 output classes | The constructor had `nn.Conv2d(3, 48, ...)` and `nn.Linear(1024, 11)` hardcoded instead of taking `in_channels`/`num_classes` as arguments | Added `in_channels`/`num_classes` as required arguments and passed them through to the first conv layer and the final linear layer | `fb904570dda45a463bb55d00475e847c0cf9ca2a` |
| 12 | models.py | `AlexNet` and `VGG16` crashed with a shape error in the classifier for any image size besides the one they were originally built for | The flattened feature size was hardcoded for a 64x64 input | Added `_get_flatten_dim`, which runs a dummy forward pass to work out the correct flatten size automatically | `fd004bc34b5c71d7f369a20cf3aa60298d34f9ee` |
| 13 | models.py | `ResNet18` always returned `None`, which crashed every loss calculation downstream with a `TypeError` | `forward` called `self.classifier(out)` but never returned the result | Added `return out` after computing the classifier's output | `48025e71bee7bfb326d1a6f4135827667279164b` |
| 14 | fit.py | `benchmark.py` and `predict.py` couldn't report precision, recall, or F1 — only loss and accuracy were ever calculated | `evaluate` only returned a loss/accuracy pair, with no way to get the macro-averaged metrics the benchmark report needed | `evaluate` now collects all predictions and labels and returns a full metrics dictionary using `sklearn.metrics` | `747a4eeb59eb79e9736870e179d54bb1c95323f0` |
| 15 | fit.py | Trained weights were lost the moment a training run ended, so there was no way to reuse a model in `predict.py` or for transfer learning | `Trainer` had no way to save checkpoints at all | `Trainer.__init__` now takes `save_model`/`model_path`, and `fit()` saves a checkpoint whenever validation accuracy improves, plus returns the per-epoch loss history | `ed2cd4cb4a6f972c2617f830e477d568303e4fbc` |
| 16 | data.py / models.py / train.py | The setup was rigid — changing the dataset, model, or any hyperparameter meant editing source code directly | `train.py` wired the data, model, and optimizer together by hand, with the dataset/model choice and hyperparameters baked straight into the code | Added `config.json` and `pipeline.py` so every experiment setting now comes from the config file instead | `9bc07b6333f20b56f6f229eace83972a94f16ee3` |
