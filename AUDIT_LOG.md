# Incident Audit Log — Operation Cyber-Histology

Every row below documents one bug, corruption, or anti-pattern found in the recovered codebase, the fix applied, and the commit that carries it. The `Git Commit Hash` column is filled in after the corresponding fix commits exist — run `git log --oneline` and match commit subjects against the table to update it yourself if you're applying these patches out of order.

## Section 1 — Pipeline Reconstruction

| File | Problem (Manifestation) | Root Cause | Correction Implemented | Git Commit Hash |
|------|--------------------------|------------|--------------------------|------------------|
| fit.py | Training loss never decreases; model doesn't learn | `train_one_epoch` never called `optimizer.zero_grad()`, so gradients accumulated across every batch instead of being reset each step, corrupting the update direction | Added `self.optimizer.zero_grad()` before `loss.backward()` | `<pending>` |
| fit.py | Latent footgun; shadows a Python builtin | The batch-count accumulator was named `sum`, shadowing the builtin `sum()` for the rest of the function's scope | Renamed to `total` | `<pending>` |
| data.py | `FileNotFoundError` on startup for every dataset | Loader built the path as `{data}_data.pt`; the recovered files on the backup are named `{data}.pt` | Corrected the f-string pattern | `<pending>` |
| data.py | Validation accuracy tracks training accuracy suspiciously closely; benchmark numbers inflated | The validation subset was sliced from the same full training tensor also handed to the training `DataLoader` in full, so validation rows were also being trained on | Replaced the slice with `torch.utils.data.random_split`, producing disjoint train/val subsets, seeded for reproducibility | `<pending>` |
| data.py | Inconsistent tensor rank feeding `CrossEntropyLoss` depending on dataset | Some datasets ship labels as `(N, 1)`; used raw without normalizing, giving train/test tensors inconsistent rank | Added `.squeeze(1)` to both label tensors before dataset construction | `<pending>` |
| data.py | Pipeline required hardcoded `CHANNELS`/`NUM_CLASSES` config keys that broke on every dataset except the one they were tuned for | Model shape metadata (channels, class count, resolution) was expected to be manually maintained in config instead of read from the data itself | `get_loaders` now derives `num_channels`, `num_classes`, `input_size` from the loaded tensors and returns them | `<pending>` |
| models.py | Deeper `VGGBlock` stages (3-conv) crash with channel-mismatch errors on the 2nd/3rd conv | `current_in_channels` was never updated after the first conv layer inside the loop, so every subsequent conv used the block's input channel count instead of the previous layer's output | Track and update `current_in_channels` per iteration | `<pending>` |
| models.py | VGG's 1x1 tail convolutions grow spatial dimensions instead of preserving them, breaking the downstream fixed-size flatten | Padding was hardcoded to `1` for every kernel size, including the 1x1 config-C tail convs, which should use `0` padding | Padding is now derived from kernel size (`0` for 1x1, `1` otherwise) | `<pending>` |
| models.py | Every model trains as if there were no nonlinearity at all; loss plateaus immediately, accuracy stuck near random chance | A module-level `activation_str = "Identity"` placeholder was hardcoded and never overridden by `config["ACTIVATION"]`; `ResNet18` referenced this global directly, `VGGBlock`/`AlexNet`/`VGG16` had `nn.ReLU` hardcoded outright, ignoring the config value either way | Removed the global; every model now reads `activation_str` from `kwargs` and instantiates the real activation class | `<pending>` |
| models.py | `AlexNet` can only be constructed for exactly 3 input channels and 11 classes | Constructor hardcoded `nn.Conv2d(3, 48, ...)` and `nn.Linear(1024, 11)` instead of accepting `in_channels`/`num_classes` | Added `in_channels`/`num_classes` as required constructor params, threaded through to the first conv and final linear layer | `<pending>` |
| models.py | `AlexNet`/`VGG16` raise a matrix-shape error in the classifier head for any input resolution other than the one the hardcoded `nn.Linear(2048, ...)` was tuned for | Flattened feature dimension was hardcoded assuming a fixed 64x64 input | Added `_get_flatten_dim`, which runs a dummy forward pass through `self.features` to infer the correct flatten size at construction time | `<pending>` |
| models.py | `ResNet18` always returns `None`; every downstream loss computation raises a `TypeError` | `forward` called `self.classifier(out)` as a bare statement instead of returning its result | Added `return out` after computing the classifier output | `<pending>` |
| fit.py | `benchmark.py`/`predict.py` can't report precision/recall/F1, only loss and accuracy were ever computed | `evaluate` returned a loss/accuracy tuple with no way to compute the macro-averaged metrics the benchmark report requires | `evaluate` now collects all predictions/labels and returns a metrics dict via `sklearn.metrics` | `<pending>` |
| fit.py | Every training run discarded its trained weights the moment the process exited; no way to reuse a trained model for `predict.py` or transfer learning | `Trainer` had no checkpointing mechanism at all | `Trainer.__init__` gains `save_model`/`model_path`; `fit()` saves a checkpoint whenever validation accuracy improves and returns per-epoch loss history | `<pending>` |
| data.py / models.py / train.py | Rigid infrastructure: dataset, model, and hyperparameters could only be changed by editing source | `train.py` wired data/model/optimizer together ad hoc, with dataset/model choice and hyperparameters baked into code | Introduced `config.json` and `pipeline.py`; every experiment parameter now flows through config | `<pending>` |
| (repo root) | No reproducible dependency manifest; recovered environment had no way to reinstall the correct packages | The environment-wipe destroyed the original requirements file; nothing recovered from the drive cache | Hand-written `requirements.txt` listing exactly what the code imports | `<pending>` |
